## 🔥 CI/CD 개념 및 도구 비교

### CI/CD 개념

#### CI (Continuous Integration, 지속적 통합)

- 정의
  - 개발자가 작업한 코드를 메인 저장소에 자주 통합하고, 
  - 통합 시 자동으로 빌드, 테스트, 정적 분석을 수행해 문제를 조기에 발견하는 프로세스

- 목표
  - 코드 충돌 및 버그를 빠르게 발견
  - 모든 팀원이 안정적인 코드 기반에서 개발 가능
  - 통합 과정에서 반복되는 테스트/검사를 자동화하여 생산성 향상

- 자동화 범위
  - 빌드
  - 유닛 테스트
  - 정적 분석 (Lint)
  - 코드 스타일 검사

#### CD (Continuous Deployment, 지속적 배포)

- 정의
  - CI 단계가 완료되면, 사람의 개입 없이 자동으로 배포 환경까지 전달하는 프로세스
- 목표
  - 빠르고 안정적인 배포
  - 배포 과정에서 발생하는 수동 작업 감소
  - QA, 베타, 프로덕션 등 여러 트랙으로의 일관된 배포
- 자동화 범위
  - 앱 서명 (Signing)
  - APK/AAB 생성
  - 테스트/프로덕션 트랙 업로드

### GitHub Actions vs Jenkins 비교

#### GitHub Actions

- 특징

  - GitHub에 통합된 CI/CD 서비스

  - 별도 서버 관리 필요 없음 (GitHub에서 관리)

  - YAML 파일로 워크플로우 정의

  - 개인/소규모 팀에게 적합
  - Public 저장소 무료, Private 저장소는 무료 사용량 제공 후 초과 시 종량제

- 장점

  - GitHub와의 완벽한 통합
  - 쉬운 설정
  - 낮은 진입 장벽

- 단점

  - GitHub 서비스에 종속
  - 복잡한 내부 시스템과의 연동은 어려울 수 있음

#### Jenkins

- 특징
  - 직접 서버 구축 후 운영해야함
  - 플러그인이 다양하고 확장성 높음
  - 기업 환경에서 널리 사용됨
  - 소프트웨어 무료, 서버 운영 비용 발생
- 장점
  - 높은 유연성
  - 강력한 커스터마이징
  - 내부망 연동 용이
- 단점
  - 높은 관리 부담
  - 보안 및 업데이트 관리 필요
