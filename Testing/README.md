## 🔥 테스트 코드

### 테스트 코드 개요

> 테스트 코드는 앱의 실제 기능 코드가 예상대로 정확하게 작동하는지 검증하기 위해 작성하는 별도의 코드

<br>

특정 함수, 클래스, 또는 전체 시스템의 동작을 실행하여

`이러한 입력이 주어졌을 때, 이러한 출력을 반환해야 한다`는 개발자의 의도나 요구사항을 검증한다

- 테스트 대상 : 비즈니스 로직, 데이터 처리, UI 상호작용, 컴포넌트 간의 연동 등이 될 수 있다

- 검증 방식 : Assertion 기법을 사용하여 실제 결과와 예상 결과를 비교한다

  (예: Junit의 assertEquals)

<br>

### 테스트 코드 작성의 필요성

- 버그 조기 발견 및 비용 절감

  - 버그는 개발 단계 초기에 발견할수록 수정 비용이 많이 낮아진다

    테스트 코드는 코드가 배포되기 전에 문제를 식별하게 해준다

  - 사람이 놓치기 쉬운 복잡한 엣지 케이스나, 대량의 데이터 처리 로직을 빠르고 정확하게 검증할 수 있다

- 안전한 코드 변경

  - 새로운 기능을 추가하거나 기존 코드를 수정했을 때, 이전에 잘 작동하던 기능이 오작동할 수 있다
  - 테스트 코드가 있다면, 변경 후 전체 테스트를 실행하여 안정성을 확보한다

- 클린 아키텍처 유지

  - 테스트 코드를 작성하려면 코드가 테스트하기 쉽도록 작성되어야 한다
  - 이는 낮은 결합도와 높은 응집도를 가진 좋은 코드를 유도한다

<br>

### 테스트 코드 장점

- 안전한 리팩토링과 유지보수 용이성
  - 테스트 코드는 리팩토링의 안전망 역할을 한다
  - 기존 코드를 개선하거나 구조를 변경할 때, 테스트가 통과하면 변경을 진행할 수 있다
- 실행 가능한 문서 역할
  - 테스트 코드는 해당 기능이 어떻게 사용되어야 하고, 어떤 결과를 기대하는지에 대한 사용 예시를 제공한다
- 개발 생산성 향상
  - 장기적으로 버그 수정 및 재작업 시간이 감소하여 개발 속도와 안정성을 높인다
